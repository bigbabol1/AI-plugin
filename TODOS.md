## P2: Expose AI Hub as MCP Server

**What:** Add an MCP server endpoint to AI Hub so external AI clients (Claude Desktop, Cline, Continue.dev, any MCP-compatible tool) can connect and use AI Hub's full tool registry (home control + web search + custom MCP tools).

**Why:** Completes the "universal AI layer" vision. Without this, AI Hub is only accessible through HA's voice/text interface. With this, any AI client on your network can control your home through AI Hub.

**Example use case:** Open Claude Desktop on laptop, say "turn off the living room lights" → Claude Desktop connects to AI Hub's MCP server → executes via HA.

**Context:** HA's built-in `mcp_server` already exposes raw HA entities. AI Hub's MCP server would expose the richer layer: search tools, custom MCP tools, skills. Different from and complementary to HA's built-in server.

**Where to start:** Add `mcp_server.py` to the integration. Register an HTTP endpoint in HA. Implement MCP protocol (list_tools, call_tool) using the `mcp` Python SDK's server-side API.

## P3: Semantic Long-Term Memory

**What:** Persist conversation facts across HA restarts using embeddings stored in `.storage`. Currently conversation history resets on every HA restart (sliding window is in-memory only).

**Why:** Users want the AI to remember preferences and facts over time. "Remember I prefer the bedroom at 68°F at night" should survive a reboot.

**Technical approach:** Add a `SemanticContextManager` that extends the abstract `ContextManager` base class. Store embeddings of key facts using Ollama's embedding endpoint (e.g., `nomic-embed-text`). On each query, retrieve top-N relevant facts by cosine similarity and inject them into the system prompt.

**Context:** The `ContextManager` is designed as an abstract class specifically to support this swap. No orchestrator refactoring needed — just a new implementation of the interface.

**Where to start:** `context_manager.py` abstract base class → `SemanticContextManager` in `context_manager_semantic.py`. Use `aiofiles` + JSON for `.storage` backing. Ollama embedding API: `POST /api/embeddings`.

**Effort:** L human / M with CC | **Priority:** P3 | **When:** Phase 3 (after MCP server)
